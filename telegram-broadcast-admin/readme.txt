README: Сервис массовой рассылки сообщений

1. Обзор

Данный сервис позволяет пользователям веб-интерфейса массово рассылать сообщения (текст + медиа) другим пользователям Telegram. Основной принцип — копирование уже существующего сообщения.


2. Процесс работы

2.1. Создание исходного сообщения и получение метаданных

Пользователь создает сообщение: Пользователь вводит текст и/или загружает медиафайл через веб-интерфейс.
Отправка системному пользователю: Веб-сервис отправляет это сообщение (текст+медиа) системному аккаунту/боту (например, самому себе в личный чат или в специально выделенный чат).
Получение fromChatId и messageId: Telegram API возвращает message_id только что отправленного сообщения. fromChatId — это ID чата, куда было отправлено исходное сообщение (системный аккаунт). Эти метаданные сохраняются.
Цель: Получить file_id для медиа и message_id для дальнейшего копирования.

2.2. Инициирование массовой рассылки

Выбор получателей: Пользователь выбирает список получателей (ID пользователей Telegram).
Запуск рассылки: Пользователь нажимает кнопку "Отправить".

2.3. Массовое копирование сообщений

Вызов TelegramHttpService.sendcopyMessage(): Сервис получает сохраненные fromChatId и messageId (из шага 2.1) и список recipientIds.
Цикл копирования: Метод sendcopyMessage итерируется по recipientIds.
Telegram API copyMessage: Для каждого recipientId вызывается copyMessage с параметрами:
chat_id: ID текущего получателя.
from_chat_id: ID системного чата (где находится исходное сообщение).
message_id: ID исходного сообщения.

2.4. Результат рассылки

Сервис возвращает список результатов (SendResult), указывающий для каждого userId:
200 (успех) или {код ошибки}/403 (неудача),
userId получателя.

3. Ключевые преимущества

Эффективность: Используется Telegram API copyMessage, избегая повторной загрузки медиа.
Простота: Логика основана на стандартном функционале Telegram.
